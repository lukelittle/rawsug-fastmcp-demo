What is the Model Context Protocol?

MCP is an open standard for connecting AI agents to external systems.

üîå Think of it as a universal adapter that lets any AI agent talk to any tool or data source without custom integration code.

  • Thousands of MCP servers built in just months

How MCP Works: Architecture

Client-server architecture using JSON-RPC 2.0

Three Core Primitives:

  • Resources GET - Load information into LLM context (files, database records, API responses)
  • Tools POST - Execute code with side effects (create tickets, deploy code, send messages)
  • Prompts TEMPLATE - Reusable templates for common tasks

Clean request-response pattern with JSON-RPC 2.0 messaging

The Communication Flow

Building with FastMCP

Decorator-based Python framework that makes building MCP servers feel like writing FastAPI

What FastMCP handles for you:

  • JSON-RPC protocol details
  • Connection management
  • Automatic JSON Schema generation from type hints
  • Serialization/deserialization
  • Error handling
  • Both stdio and SSE transports

Simple FastMCP Server

A complete, working MCP server in 10 lines:

from fastmcp import FastMCP

# Initialize server
mcp = FastMCP("My First Server")

# Define a tool
@mcp.tool()
def add_numbers(a: int, b: int) -> int:
    """Add two numbers together"""
    return a + b

# Run it
if __name__ == "__main__":
    mcp.run()

That's it! No boilerplate. No protocol details. Just your logic.

Real Example: GitHub Integration

from fastmcp import FastMCP
from github import Github

mcp = FastMCP("GitHub Integration")
gh = Github(os.getenv("GITHUB_TOKEN"))

@mcp.tool()
def get_repository_info(owner: str, repo: str) -> dict:
    """Get information about a GitHub repository."""
    repository = gh.get_repo(f"{owner}/{repo}")
    
    return {
        "name": repository.name,
        "stars": repository.stargazers_count,
        "forks": repository.forks_count,
        "language": repository.language
    }

Now any MCP client can query GitHub!

Advanced FastMCP Features

  • Context Injection - Send progress updates to clients during long operations
  • Sampling - Request LLM completions from the client (no API keys needed)
  • Elicitation - Request additional info mid-operation
  • Filesystem Roots - Security boundaries for file access
